Function -VAU_xfa Functions Script Package Keyword Index Function Index Home
    Signature :    -VAU_xfa(sMode)
    Source :    vau.itp(49)

Description

Computes UASR feature vector files from the signal files listed in <uasr.flist.all>.

Parameters

   string    sMode   Operation mode, one of the following

"pfv" UASR primary features, see -CMD_DB_pfv
"sfv" 1) UASR secondary features, see -CMD_DB_sfv
"lfv" 1) UASR least significant secondary features, see -CMD_DB_lfv
"rnd" Random features, see -CMD_DB_rnd

1) Mode requires a feature information file <uasr.dir.model>/feainfo.object as generated by the UASR HMM training tool HMM.xtp trn.

Return value

0 if successful, -1 otherwise.

Function -CMD_DB_pfv Functions Script Package Keyword Index Function Index Home
    Signature :    -CMD_DB_pfv
    Source :    vau.itp(199)

Description

Command handler for DB.xtp: Computes UASR primary feature vector files from signal files listed in <uasr.flist.all>. The primary feature type can be configured through <uasr.pfa> (default is "MEL"). Output files are written to <uasr.dir.data>/common/fea/pfv_<uasr.pfa.dim>. The output file format is plain text, each line containing one feature vector of white space separated floats.

Example

 cd $UASR_HOME-data/vau
 DB.xtp pfv common/info/featureanalysis.cfg

See also

   -CMD_DB_sfv
   -CMD_DB_lfv

Function -CMD_DB_lfv Functions Script Package Keyword Index Function Index Home
    Signature :    -CMD_DB_lfv
    Source :    vau.itp(252)

Description

Command handler for DB.xtp: Computes UASR least significant secondary feature vector files files from signal files listed in <uasr.flist.all>. The least significant secondary featues comprise the vector components discarded by the PCA. The primary feature type can be configured through <uasr.pfa> (default is "MEL"). Output files are written to <uasr.dir.data>/common/fea/lfv_<dim> where dim denotes the number of least significant features (depends on the primary and secondary feature extraction settings). The output file format is plain text, each line containing one feature vector of white space separated floats.

Note: A feature information file <uasr.dir.model>/feainfo.object as generated by the UASR HMM training tool HMM.xtp trn is required.

Example

 cd $UASR_HOME-data/vau
 DB.xtp lfv common/info/featureanalysis.cfg

See also

   -CMD_DB_pfv
   -CMD_DB_sfv
Function -CMD_DB_rnd Functions Script Package Keyword Index Function Index Home
    Signature :    -CMD_DB_rnd
    Source :    vau.itp(269)

Description

Command handler for DB.xtp: Generates 10-dimensional random feature vector files of random length in the interval [250...350] for the files listed in <uasr.flist.all>. Output files are written to <uasr.dir.data>/common/fea/rnd_10. The output file format is plain text, each line containing one feature vector of white space separated floats.

The function is intended for debugging the GMM classifier. As there is nothing to be learned from the generated features the classifier correctness is expected to be equal to mere guessing (1/#speakers).

Note: The <uasr.flist.all> file list is used.
